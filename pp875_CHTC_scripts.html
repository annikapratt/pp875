03 Data and Software Setup

Miniconda installation

Working with software on the CHTC can be a bit of a headache, but conda makes this process much less painful. The first step of this analysis is to install miniconda3 on the CHTC. I followed Option B of this tutorial to learn how to install miniconda - see the commands below.

# type these in the terminal when you are connected to the CHTC and in your home directory

# run the script that you just downloaded to install miniconda3

After running that last command, I accepted all default options, which activated the base conda environment. I know the base environment is active because before my username and promt it says (base).

Creating a git repository

Now that git is properly installed and configured on the CHTC, it is time to make a git repository. Software Carpentries has a really great lesson for getting started with git. I refer back to this whenever I forget a command or need to re-initialize git on a new machine.

# make a new directory on the CHTC
mkdir pp875

# tell git to start tracking that folder
git init

# run this command to see when something needs to be added, committed, or pushed
git status

git add # tells git to start tracking the file(s)
git commit # actually records the changes
git push # sends those changes to GitHub (they are no longer locally on one machine)

Next we want our local git repo to communicate with github. First step is to create a github account. Then click Repositories > New to create a new repository named pp875. To connect this repository with my local repo, I clicked the green ‘code’ button and copied the URL that shows up there. Next, I went back to my CHTC terminal:

# this is letting the two repositories (GitHub and CHTC) connect
# The URL here is what I copied from GitHub
git remote add origin https://github.com/annikapratt/pp875.git

# I tested if this worked by creating a file
touch file.txt

# checking the status, adding, committing, and pushing the update
# after running the commands below I checked GitHub and saw that the file appeared there
git status 
git add file.txt
git commit -m "adding a file to test git connection"
git push

The last setup step is to copy the class .gitignore into our git repository. To do this, I just when to the github, copied the text of the .gitignore, and pasted it into a new text file called .gitignore in my pp875 directory on the CHTC. The gitignore is essential for telling git what not to track (typically things that will not change - data, software, etc.).

04 Gene Tree Estimation

The data started with 13,288 genes which were filtered down to the 8,739 we are using here. We are using raxml to estimate gene trees based on those 8,739 alignments. Before running the whole thing, I tested the command on just 10 genes to make sure it all works.

# this is in the data/Wheat_Relative_History_Data_Glemin_et_al directory

# unzip IndividualAlignments_OneCopyGenes.zip
unzip IndividualAlignments_OneCopyGenes.zip

# make a subfolder for the smaller dataset
mkdir OneCopyGenes-trimmed

# move to OneCopyGenes
cd OneCopyGenes

# copy 10 genes to the trimmed folder
ls | head -n10 | xargs -I {} cp "{}" ../OneCopyGenes-trimmed

The subfolder now contains the first 10 alignments

# move to the code folder to run the raxml script
cd ../../code
./04-raxml.sh

We can see the results for each gene in the results/RAxML folder. Running 10 genes in the terminal wasn’t bad, but to run the whole dataset we’ll want to break the dataset into several chunks and submit jobs.

# move into your data/Wheat_Relative_History_Data_Glemin_et_al folder

num_dirs=20

# Create destination directories
for i in $(seq 1 $num_dirs); do
    mkdir -p "OneCopyGenes_$i"
done

# Copy files evenly
i=1
for file in OneCopyGenes/*; do
    cp "$file" "OneCopyGenes_$i/"
    i=$(( (i % num_dirs) + 1 ))
done
# run the 20 jobs
# docker container is raxml-ng version 2.0.0
condor_submit 04-job-raxml.sub

The results ended up in 20 separate folders in the code folder, so we have to move these folders to the results/RAxML folder

from Bio import AlignIO
from Bio.AlignIO.PhylipIO import SequentialPhylipWriter

data_path = "../data/Wheat_Relative_History_Data_Glemin_et_al/"
concat_file = "triticeae_allindividuals_OneCopyGenes.fasta"

fasta_file = data_path+concat_file
phylip_file = "../results/10-triticeae_allindividuals_OneCopyGenes.phylip"

# Load the alignment
with open(fasta_file, "r") as f_in:
    alignment = AlignIO.read(f_in, "fasta")

# Find the length of the longest sequence ID to prevent truncation
max_id_len = max(len(record.id) for record in alignment)

# Write out the alignment
with open(phylip_file, "w") as f_out:
    writer = SequentialPhylipWriter(f_out)
    writer.write_alignment(alignment, id_width=max_id_len + 3) # 3 additional padding
# convert fasta to phylip with script above
